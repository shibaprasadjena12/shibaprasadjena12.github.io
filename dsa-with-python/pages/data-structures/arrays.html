Arrays (Lists)

Fundamental collection data structure for storing multiple elements

What is an Array?

An array is a fundamental data structure that stores a collection of elements of the same type in contiguous memory locations. In Python, arrays are typically implemented as lists, which are more flexible and can store elements of different types.

Arrays provide fast access to elements through their index position, making them one of the most widely used data structures in programming.

Key Characteristics

  • Index-based: Elements are accessed using their position (0-indexed in Python)
  • Contiguous memory: Elements are stored in adjacent memory locations
  • Fixed size: Traditional arrays have fixed size (Python lists are dynamic)
  • Homogeneous: Traditional arrays store same type (Python lists can store mixed types)
  • Random access: O(1) time to access any element

Basic Operations

1. Creating Arrays/Lists

# Creating empty list
arr = []
arr = list()

# Creating list with initial values
arr = [1, 2, 3, 4, 5]

# Creating list with mixed types
arr = [1, "hello", 3.14, True]

# List comprehension
arr = [i for i in range(5)]  # [0, 1, 2, 3, 4]

# Creating list of specific size
arr = [0] * 5  # [0, 0, 0, 0, 0]

2. Accessing Elements

arr = [10, 20, 30, 40, 50]

# Access by index (0-indexed)
print(arr[0])      # Output: 10
print(arr[2])      # Output: 30

# Negative indexing
print(arr[-1])     # Output: 50 (last element)
print(arr[-2])     # Output: 40 (second last element)

# Slicing
print(arr[1:4])    # Output: [20, 30, 40]
print(arr[:3])     # Output: [10, 20, 30]
print(arr[2:])     # Output: [30, 40, 50]

3. Modifying Elements

arr = [1, 2, 3, 4, 5]

# Update single element
arr[0] = 10
print(arr)  # Output: [10, 2, 3, 4, 5]

# Update slice
arr[1:3] = [20, 30]
print(arr)  # Output: [10, 20, 30, 4, 5]

4. Adding Elements

arr = [1, 2, 3]

# Append (add to end)
arr.append(4)
print(arr)  # Output: [1, 2, 3, 4]

# Insert at specific position
arr.insert(1, 10)
print(arr)  # Output: [1, 10, 2, 3, 4]

# Extend (add multiple elements)
arr.extend([5, 6])
print(arr)  # Output: [1, 10, 2, 3, 4, 5, 6]

5. Removing Elements

arr = [1, 2, 3, 4, 5]

# Remove by value
arr.remove(3)
print(arr)  # Output: [1, 2, 4, 5]

# Remove by index (pop)
arr.pop()   # Removes last element
print(arr)  # Output: [1, 2, 4]

arr.pop(0)  # Removes first element
print(arr)  # Output: [2, 4]

# Clear entire list
arr.clear()
print(arr)  # Output: []

6. Searching Elements

arr = [10, 20, 30, 40, 50]

# Find index of element
index = arr.index(30)
print(index)  # Output: 2

# Check if element exists
if 40 in arr:
    print("Element found")

# Count occurrences
arr = [1, 2, 2, 3, 2, 4]
count = arr.count(2)
print(count)  # Output: 3

7. Sorting and Reversing

arr = [3, 1, 4, 1, 5, 9]

# Sort (modifies original)
arr.sort()
print(arr)  # Output: [1, 1, 3, 4, 5, 9]

# Reverse (modifies original)
arr.reverse()
print(arr)  # Output: [9, 5, 4, 3, 1, 1]

# Sorted (returns new list)
arr = [3, 1, 4, 1, 5]
sorted_arr = sorted(arr)
print(sorted_arr)  # Output: [1, 1, 3, 4, 5]
print(arr)         # Original unchanged: [3, 1, 4, 1, 5]

Complexity Analysis

Time Complexity:

Access: O(1) - Direct access using index

Search: O(n) - Linear search through elements

Insertion: O(n) - May require shifting elements

Deletion: O(n) - May require shifting elements

Append: O(1) amortized - Adding to end

Space Complexity:

O(n) - Where n is the number of elements stored

Common Problems

Problem 1: Find Maximum Element

def find_max(arr):
    """
    Find the maximum element in an array.
    Time Complexity: O(n)
    Space Complexity: O(1)
    """
    if not arr:
        return None
    
    max_val = arr[0]
    for num in arr:
        if num > max_val:
            max_val = num
    return max_val

# Using built-in
max_val = max(arr)

# Test
arr = [3, 1, 4, 1, 5, 9]
print(find_max(arr))  # Output: 9

Problem 2: Rotate Array

def rotate_array(arr, k):
    """
    Rotate array k times to the right.
    Time Complexity: O(n)
    Space Complexity: O(1)
    """
    n = len(arr)
    k = k % n  # Handle k > n
    
    # Reverse the entire array
    arr.reverse()
    # Reverse first k elements
    arr[:k] = arr[:k][::-1]
    # Reverse remaining elements
    arr[k:] = arr[k:][::-1]
    
    return arr

# Test
arr = [1, 2, 3, 4, 5]
print(rotate_array(arr, 2))  # Output: [4, 5, 1, 2, 3]

Problem 3: Remove Duplicates

def remove_duplicates(arr):
    """
    Remove duplicates from sorted array.
    Time Complexity: O(n)
    Space Complexity: O(1)
    """
    if len(arr) <= 1:
        return arr
    
    j = 0
    for i in range(1, len(arr)):
        if arr[i] != arr[j]:
            j += 1
            arr[j] = arr[i]
    
    return arr[:j+1]

# Test
arr = [1, 1, 2, 2, 3, 4, 4, 5]
print(remove_duplicates(arr))  # Output: [1, 2, 3, 4, 5]

When to Use Arrays

  • When you need fast random access to elements
  • When you need to store a collection of elements
  • When memory is limited (more efficient than linked lists)
  • When you're working with sorted data
  • As the foundation for many other data structures

üí° Key Takeaways

Remember These Points:

  • Arrays are 0-indexed - first element is at index 0
  • Negative indices work from the end: arr[-1] is the last element
  • Slicing creates a NEW array, doesn't modify original
  • append() is O(1) amortized, insert() at position is O(n)
  • Use lists for dynamic sizing, traditional arrays for fixed
  • List comprehension is Pythonic and efficient

Common Mistakes to Avoid:

  • Forgetting 0-based indexing - arr[n] for n elements is out of bounds
  • Not checking if array is empty before accessing elements
  • Assuming slicing modifies the original array
  • Using inefficient O(n) insertion - use append() at end instead
  • Creating copies when you don't need to - wastes memory
  • Not handling negative indices properly in loops
