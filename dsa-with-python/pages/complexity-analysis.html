Complexity Analysis

Understanding Time and Space Complexity of Algorithms

What is Complexity Analysis?

Complexity analysis measures how an algorithm's performance changes as input size grows. It helps us choose the right algorithm for a problem and predict performance at scale.

Big-O Notation

Big-O describes the worst-case scenario - the maximum time or space an algorithm might use.

Common Time Complexities (from fastest to slowest)

Time Complexity Rules

# Drop Constants
# O(2n) ‚Üí O(n)
def example1(n):
    for i in range(2*n):  # 2n operations
        print(i)

# Drop Lower Order Terms
# O(n¬≤ + n) ‚Üí O(n¬≤)
def example2(n):
    for i in range(n):      # n¬≤ operations
        for j in range(n):
            print(i, j)
    for i in range(n):      # n operations
        print(i)

# Multiple Inputs
# O(m*n)
def example3(m, n):
    for i in range(m):
        for j in range(n):
            print(i, j)

# Add Complexities (Sequential)
# O(n + m)
def example4(n, m):
    for i in range(n):      # O(n)
        print(i)
    for i in range(m):      # O(m)
        print(i)

Space Complexity

Space complexity measures the amount of extra memory an algorithm uses.

  • O(1): Constant space (in-place algorithms)
  • O(n): Linear space (arrays, lists)
  • O(log n): Logarithmic space (recursion depth)
  • O(n¬≤): Quadratic space (2D arrays)

Big-O vs Other Notations

  • Big-O (O): Upper bound - worst case
  • Big-Omega (Œ©): Lower bound - best case
  • Big-Theta (Œò): Tight bound - average case

Practical Tips

  • Nested loops multiply complexities: O(n) * O(n) = O(n¬≤)
  • Sequential loops add complexities: O(n) + O(m) = O(n + m)
  • Always consider best, average, and worst cases
  • Space includes both input and auxiliary space

üí° How to Analyze Your Code

  1. Identify Loops: Count nested loops. Each level of nesting multiplies the complexity.
  2. Check Operations: Are operations inside loops O(1) or do they take time?
  3. Recursion: For recursive code, multiply calls √ó depth.
  4. Data Structures: Different operations have different complexities (e.g., list.insert() is O(n)).

üéØ Algorithm Complexity Cheat Sheet

Data Structure Operations:

  • Array Access: O(1)
  • Array Search: O(n)
  • Array Insert/Delete: O(n)
  • Hash Table Lookup: O(1) average
  • Binary Search Tree Search: O(log n) average
  • Heap Push/Pop: O(log n)

Sorting Algorithms:

  • Bubble/Selection/Insertion: O(n¬≤)
  • Merge Sort: O(n log n)
  • Quick Sort: O(n log n) average, O(n¬≤) worst
  • Heap Sort: O(n log n)
  • Counting Sort: O(n + k)

‚ùì Common Interview Questions

# Q1: What's the complexity of this?
def mystery(n):
    for i in range(n):           # O(n)
        for j in range(n):       # O(n)
            print(i, j)
# Answer: O(n¬≤) - nested loops multiply

# Q2: And this?
def mystery2(n):
    for i in range(n):           # O(n)
        print(i)
    for j in range(n):           # O(n)
        print(j)
# Answer: O(n) - sequential loops add, then simplify

# Q3: How about this recursive function?
def factorial(n):                # Called n times
    if n == 1:                   # O(1)
        return 1
    return n * factorial(n - 1)  # n times recursion
# Answer: O(n) - n recursive calls, each does O(1) work

# Q4: What about binary search?
def binary_search(arr, target):
    low, high = 0, len(arr) - 1
    while low <= high:           # Divides by 2 each time
        mid = (low + high) // 2
        if arr[mid] == target:
            return mid
        elif arr[mid] < target:
            low = mid + 1
        else:
            high = mid - 1
    return -1
# Answer: O(log n) - halves search space each iteration
